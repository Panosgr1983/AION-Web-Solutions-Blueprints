# AION Playbooks

## What is a Playbook?
A playbook is a step-by-step methodology for a specific domain or task. Unlike a checklist (what to do), a playbook explains how to think about it, how to approach it, and how to execute it consistently.

## Playbook Format (Template)

```
# [Title] Playbook

## Objective
What does this playbook achieve?

## Prerequisites
What must be in place before starting?

## Phases
1. [Phase 1] — [Timeline]
   Goal:
   Steps:
   - [ ] Step 1
   - [ ] Step 2
   Deliverable:

2. [Phase 2] — [Timeline]
   ...

## Decision Points
At each phase, what decisions need to be made and who makes them.

## Common Mistakes
What to avoid.

## Success Criteria
How do you know this playbook was executed successfully?
```

## Playbook Principles
1. **Start with why** — understanding > following steps
2. **Make decisions explicit** — every phase has a decision point
3. **Document mistakes** — they're more valuable than successes
4. **Keep them alive** — update after every project
5. **Don't over-engineer** — a playbook should fit on 2-3 pages
